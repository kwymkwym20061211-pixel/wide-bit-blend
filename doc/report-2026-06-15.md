## 1. テスト方法の拡張：PractRand パイプテスト

### 背景

`mixer_test.h` の内製テストスイートは以下4項目を測定する。

| テスト | 内容 |
|---|---|
| 雪崩効果 (Avalanche) | 1bit変化で出力の何bitが反転するか（理想: 50%） |
| ビット独立性 (BIC) | 各入出力bitペアの独立性スコア（理想: 1.0） |
| 均一性 (Uniformity) | 出力の各bitが1になる確率（理想: 50%） |
| 衝突耐性 (Collision) | 出力ペア間ハミング距離の分布（理想: avg=128, σ≈8） |

これらはランダム入力に対する品質を測るが、**連続入力（カウンタ列など）に対する線形構造の残留**は検出できない。

PractRand は擬似乱数の統計的品質を検証する業界標準ツールで、ミキサーをカウンタモードPRNGとして使った際の欠陥を発見できる。

### セットアップ

`tools/prng_stdout.c` を新規作成した。ミキサーにカウンタを入力として与え、出力バイト列を stdout に流す。

```c
// 256bitカウンタを1ずつインクリメントし続ける
static inline void increment_counter(mt_block_t *c) {
    for (int i = 0; i < 4; i++) {
        c->word[i]++;
        if (c->word[i] != 0) break;
    }
}

int main(void) {
    mt_block_t counter = {{1, 0, 0, 0}};  // ゼロ固定点を回避するため1スタート
    while (1) {
        mt_block_t out = counter;
        mixer(&out);
        fwrite(out.data, 1, MT_BLOCK_SIZE, stdout);
        increment_counter(&counter);
    }
}
```

> **なぜカウンタを1スタートにするか**
> murmur ファイナライザは `0 → 0` の固定点を持つ（乗算・XORが全て0に吸収される）。
> 入力ゼロのまま流すと出力の先頭32byteが全ゼロになるため、カウンタを1から始める。

PractRand のビルドと実行コマンド:

```sh
# PractRand ビルド (一度だけ)
git clone https://github.com/tylov-fork/PractRand.git
cd PractRand
g++ -std=c++14 -O3 -Iinclude src/*.cpp src/RNGs/*.cpp src/RNGs/other/*.cpp tools/RNG_test.cpp -o RNG_test -lpthread

# テスト実行
./compiled/prng_stdout | ./PractRand/RNG_test stdin -tlmax 4G
```

## 2. 第4世代ミキサー (2026-05-14) の問題発見

### コード

```c
static void mixer_carried_murmur_2026_05_14(mt_block_t *value_ref)
{
    mt_block_t tmp = *value_ref;
    for (int i = 0; i < 4; i++)
    {
        tmp.word[i % 4] ^= tmp.word[i % 4] >> 33;
        tmp.word[i % 4] *= 0xff51afd7ed558ccdULL;
        tmp.word[i % 4] ^= tmp.word[i % 4] >> 33;
        tmp.word[i % 4] *= 0xc4ceb9fe1a85ec53ULL;
        tmp.word[i % 4] ^= tmp.word[i % 4] >> 33;
        tmp.word[(i + 1) % 4] ^= tmp.word[i % 4];
        tmp.word[(i + 2) % 4] ^= tmp.word[i % 4];
        tmp.word[(i + 3) % 4] ^= tmp.word[i % 4];
    }
    *value_ref = tmp;
}
```

### 内製テスト結果（65,536 trials）— 全PASS

```
--- Avalanche Effect ---
  avg flip rate : 0.5000  result : PASS
--- Bit Independence ---
  avg score     : 0.9992  result : PASS
--- Uniformity ---
  avg 1-rate    : 0.5000  result : PASS
--- Collision Resistance ---
  exact collisions : 0    result : PASS
--- Performance ---
  ns/call : 31.4 ns
  OVERALL: PASS
```

### PractRand 結果（64MB）— **FAIL**

```
rng=RNG_stdin, seed=unknown
length= 64 megabytes (2^26 bytes), time= 1.1 seconds
  [Low4/32]Gap-16:A     R= +1290  p =  1e-1108    FAIL !!!!!!!!
  [Low4/32]Gap-16:B     R=  +14.0 p =  2.1e-11    FAIL
  [Low4/32]BRank(12):384(1)       p~=  8.9e-22    FAIL !!
  [Low4/32]BRank(12):512(1)       p~=  6.1e-73    FAIL !!!!
  ...and 175 test result(s) without anomalies
```

### 原因分析

`[Low4/32]` プレフィックスは「出力を32bit整数として解釈し、その**下位4bit**のみを検定する」テストである。

第4世代のXOR伝播は線形操作であり、murmur ファイナライザを1回しか適用しないため、連続カウンタ（下位ビットが規則的に変化）の線形構造がXOR伝播の線形性と組み合わさり、下位ビットに統計的パターンとして残留した。

```
// XOR は線形操作 → 下位ビットの線形性が保存される
tmp.word[(i + 1) % 4] ^= tmp.word[i % 4];
tmp.word[(i + 2) % 4] ^= tmp.word[i % 4];
tmp.word[(i + 3) % 4] ^= tmp.word[i % 4];
```

内製テストはランダム入力を前提とするため、この線形残留を検出できなかった。

## 3. 第5世代ミキサー (2026-06-15) による修正

### 変更内容

ループを **1回 → 2回** に増やした。これにより各 word が murmur ファイナライザを2回通り、1回目のXOR伝播で生じた線形残留が2回目のファイナライザで完全に破壊される。

### コード

```c
static void mixer_carried_murmur_2026_06_15(mt_block_t *value_ref)
{
    mt_block_t tmp = *value_ref;

    for (int i = 0; i < 4; i++)
    {
        tmp.word[i % 4] ^= tmp.word[i % 4] >> 33;
        tmp.word[i % 4] *= 0xff51afd7ed558ccdULL;
        tmp.word[i % 4] ^= tmp.word[i % 4] >> 33;
        tmp.word[i % 4] *= 0xc4ceb9fe1a85ec53ULL;
        tmp.word[i % 4] ^= tmp.word[i % 4] >> 33;
        tmp.word[(i + 1) % 4] ^= tmp.word[i % 4];
        tmp.word[(i + 2) % 4] ^= tmp.word[i % 4];
        tmp.word[(i + 3) % 4] ^= tmp.word[i % 4];
    }
    for (int i = 0; i < 4; i++)   // ← 2回目：線形残留を破壊
    {
        tmp.word[i % 4] ^= tmp.word[i % 4] >> 33;
        tmp.word[i % 4] *= 0xff51afd7ed558ccdULL;
        tmp.word[i % 4] ^= tmp.word[i % 4] >> 33;
        tmp.word[i % 4] *= 0xc4ceb9fe1a85ec53ULL;
        tmp.word[i % 4] ^= tmp.word[i % 4] >> 33;
        tmp.word[(i + 1) % 4] ^= tmp.word[i % 4];
        tmp.word[(i + 2) % 4] ^= tmp.word[i % 4];
        tmp.word[(i + 3) % 4] ^= tmp.word[i % 4];
    }

    *value_ref = tmp;
}
```

## 5. 知見まとめ

- **内製テストの限界**: 雪崩・BIC・均一性・衝突耐性の4項目はランダム入力を前提とする。連続入力に対する線形構造の残留は検出できない。第4世代は内製テストで全PASS だったが、PractRand で即座に欠陥が露見した。
- **XORは線形**: ワード間伝播にXORを使う限り、ファイナライザ通過前の線形性がそのまま伝わる。2回ループすることで伝播後に再びファイナライザを通すことができ、線形残留を非線形変換で破壊できる。
- **ゼロ固定点の注意**: murmur系ファイナライザは `0 → 0` の自明な固定点を持つ。カウンタ入力を使う場合は1以上からスタートさせること。
- **PractRandは必須**: ビットミキサーの品質を保証するには内製テストだけでは不十分で、PractRandのような外部統計テストスイートとの併用が事実上必要である。
