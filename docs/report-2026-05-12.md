# 256bitミキサー開発レポート：第2世代 vs 第3世代

本ドキュメントは、C17標準に準拠しつつ、MurmurHash3のミキシング定数を応用して設計された256bit固定長ミキサーの設計と性能を記録するものです。

## 1. 第2世代 (2026-05-12版) 処理フロー解説

**特徴：回転コピーと二重キャリー伝播による堅牢な攪拌**

1. **分散コピー:** 入力データ（32バイト）を8個の `uint64_t` バッファ（`buf[0-7]`）へ「飛び飛び」に格納。これにより、入力の隣接ビットを異なる演算ブロックへ分散させる。
2. **垂直ミキシング (1回目):** 各バッファに対し、MurmurHash3のファイナライザ（xorshiftと乗算の組み合わせ）を適用。
3. **水平伝播 (1回目):** 各バッファの上位32bitを、隣のバッファへXORで伝播（キャリー伝播）。
4. **垂直ミキシング (2回目):** 再度、全バッファを個別にミキシングし、非線形性を強化。
5. **水平伝播 (2回目):** 再度キャリーを回し、256bit全体への拡散を確実にする。
6. **復元:** ミキシングされたバッファを回転させながら元のバイト配列へXORで書き戻す。

## 2. 第3世代 (2026-05-13版) 処理フロー解説

**特徴：依存関係の整理とパイプライン最適化による高速化**

1. **二重ループによるデータロード:** 第2世代の複雑な条件分岐を含むロードを、シンプルな二重ループ（バイト段×ブロック）に変更。コンパイラのループ展開と並列ロードを促進。
2. **シングル・パス演算:** ミキシングとキャリー伝播を「1周」に最適化。第2世代で得られた知見に基づき、1周でも十分な精度が出るよう演算の依存関係を整理。
3. **ストレートな書き戻し:** 書き戻し処理も二重ループで簡素化。CPUのパイプラインを止めないメモリアクセスを実現。
4. **最適化の肝:** ループを「ハッシュ計算」と「キャリー伝播」で明確に分離したことで、コンパイラが各ブロックの独立性を認識しやすくなり、実行速度が劇的に向上（約2倍）。

## 3. ソースコード (C17準拠)

```c
// ちょっと変わったmurmur風ミキサー。ビットの回転とキャリーを入れてみたもの。雪崩効果もある安定版。
static void mixer_carried_murmur_2026_05_12(mt_block_t *value_ref)
{
    // 借りる
    mt_block_t tmp = *value_ref;

    // uint64_tのバッファにコピー。上位32bitは0になる。
    uint64_t buf[8];
    int shift_count = 0;
    // 回転しながら飛び飛びにコピーして行く。
    for (int i = 0; i < 32; i++)
    {
        buf[i % 8] |= (uint32_t)(tmp.data[i] << shift_count);
        if (i % 8 == 7)
        {
            shift_count += 8;
        }
    }

    // murmurハッシュ風の変換を二周。キャリーの上位32bitは1個上に伝播。
    // メモ：ループをそれぞれのハッシュとキャリーで分割したところ高速になった。
    for (int i = 0; i < 8; i++)
    {
        int this_idx = i;
        buf[this_idx] ^= buf[this_idx] >> 33;
        buf[this_idx] *= 0xff51afd7ed558ccdULL;
        buf[this_idx] ^= buf[this_idx] >> 33;
        buf[this_idx] *= 0xc4ceb9fe1a85ec53ULL;
        buf[this_idx] ^= buf[this_idx] >> 33;
    }
    for (int i = 0; i < 8; i++)
    {
        buf[(i + 1) % 8] ^= buf[i] >> 32;
    }
    for (int i = 0; i < 8; i++)
    {
        int this_idx = i;
        buf[this_idx] ^= buf[this_idx] >> 33;
        buf[this_idx] *= 0xff51afd7ed558ccdULL;
        buf[this_idx] ^= buf[this_idx] >> 33;
        buf[this_idx] *= 0xc4ceb9fe1a85ec53ULL;
        buf[this_idx] ^= buf[this_idx] >> 33;
    }
    for (int i = 0; i < 8; i++)
    {
        buf[(i + 1) % 8] ^= buf[i] >> 32;
    }

    // もとのバッファの形式に書き戻す。
    for (int i = 0; i < 32; i++)
    {
        int idx = i % 8;
        tmp.data[i] ^= (int8_t)(buf[idx] >> ((i / 8) * 8)); // 回転して書き戻す
        // ここで対角線上のバイトとXORしても精度に大差ないことを確認。
    }
    // 返す
    *value_ref = tmp;
}

// ちょっと変わったmurmur風ミキサー。ビットの回転とキャリーを入れてみたもの。雪崩効果もある安定版。
static void mixer_carried_murmur_2026_05_13(mt_block_t *value_ref)
{
    // 借りる
    mt_block_t tmp = *value_ref;

    // uint64_tのバッファにコピー。上位32bitは0になる。
    uint64_t buf[8] = {0};
    // 回転しながら飛び飛びにコピーして行く。
    for (int shift = 0; shift < 4; shift++)
    {
        for (int idx = 0; idx < 8; idx++)
        {
            buf[idx] |= (uint64_t)tmp.data[shift * 8 + idx] << (shift * 8);
        }
    }

    // murmurハッシュ風の変換。
    for (int i = 0; i < 8; i++)
    {
        buf[i] ^= buf[i] >> 33;
        buf[i] *= 0xff51afd7ed558ccdULL;
        buf[i] ^= buf[i] >> 33;
        buf[i] *= 0xc4ceb9fe1a85ec53ULL;
        buf[i] ^= buf[i] >> 33;
    }
    // キャリーを伝播させる。
    for (int i = 0; i < 8; i++)
    {
        buf[(i + 1) % 8] ^= buf[i] >> 32;
    }

    // もとのバッファの形式に書き戻す。
    for (int b = 0; b < 4; b++) // バイト段
    {
        for (int idx = 0; idx < 8; idx++) // ブロック
        {
            tmp.data[b * 8 + idx] ^= (uint8_t)(buf[idx] >> (b * 8));
        }
    }

    // 返す
    *value_ref = tmp;
}
```

## 5. 設計上の知見

- **キャリー伝播が核心:** murmurラウンドだけでは8ブロックが縦方向に独立して混ざるだけ。キャリー伝播によってブロック間の横方向の情報伝達が生まれ、256bitを一塊として扱える。
- **murmurラウンド1周で十分:** 2周→1周に削減しても品質スコアに変化なし。キャリー伝播が拡散を補完している。
- **ループ構造が速度を決める:** 動的シフト・条件分岐を含むループから静的二重ループへの変換が最大の高速化要因。コンパイラの最適化が素直に効く構造が重要。
